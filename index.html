Tema 1: Elementos de Páginas Web

  1. Orígenes de la Web

  1. Web 2.0

    2. Representa la evolución de la web estática (Web 1.0) hacia una web más interactiva y colaborativa.

  2. Web 3.0

    3. La Web 3.0 se centra en la creación de una web más inteligente y personalizada, donde las máquinas pueden entender y procesar el contenido de manera similar a los humanos.

  3. W3C y los estándares

    4. La World Wide Web Consortium (W3C) es una organización que desarrolla estándares abiertos para garantizar la interoperabilidad y evolución de la web. Los estándares de la W3C incluyen HTML, CSS, y JavaScript, entre otros.

  1. Lenguajes de etiquetas

  1. SGML

    2. Standard Generalized Markup Language es un estándar para definir lenguajes de marcado utilizados en la creación de documentos. Es la base de lenguajes como HTML y XML.

  2. XML

    3. Extensible Markup Language es un lenguaje de marcado diseñado para almacenar y transportar datos. Permite a los usuarios definir sus propias etiquetas y es ampliamente utilizado para el intercambio de datos en la web.

  3. HTML

    4. HyperText Markup Language es el lenguaje de marcado estándar para crear páginas web. Define la estructura y el contenido de una página web utilizando etiquetas y atributos

  4. XHTML

    5. Extensible HyperText Markup Language es una versión de HTML que sigue las reglas de XML, asegurando que los documentos sean bien formados y compatibles con los estándares.

  5. HTML5

    6. Es la última versión de HTML y ofrece mejoras significativas en la semántica, accesibilidad y funcionalidad de las páginas web. Incluye mejor soporte para multimedia y gráficos sin necesidad de complementos adicionales.

  1. Otros lenguajes relacionados al desarrollo web

  1. CSS/CSS3

    2. Cascading Style Sheets (CSS) es un lenguaje utilizado para describir la presentación de un documento escrito en HTML o XML. CSS3 es la última versión, que incluye nuevas características como animaciones, transiciones y mejores opciones de diseño.

  2. JavaScript

    3. Lenguaje de programación que permite agregar interactividad a las páginas web. Es esencial para el desarrollo de aplicaciones web dinámicas y es compatible con todos los navegadores web modernos.

  3. Ajax

    4. Técnica de desarrollo web que permite la actualización de partes de una página web sin tener que recargarla por completo. Utiliza JavaScript y XML/JSON para comunicarse de forma asíncrona con el servidor.

  4. Tecnologías y lenguajes del lado del servidor

    5. Incluyen lenguajes como PHP, Ruby, Python, y frameworks como Node.js. Estas tecnologías se utilizan para gestionar bases de datos, autenticación de usuarios y lógica de negocios en el servidor.

  1. Herramientas de desarrollo

  1. Software WYSIWYG

    2. Son editores que permiten diseñar páginas web de manera visual, sin necesidad de escribir código manualmente. Ejemplos incluyen Adobe Dreamweaver y Microsoft Expression Web.

  2. Otras alternativas

    3. Herramienta de diseño colaborativa basada en la nube, excelente para prototipos de interfaz de usuario.

  3. Figma

    4. Herramienta de diseño colaborativa basada en la nube, excelente para prototipos de interfaz de usuario.

  4. InVisión

    5. Plataforma de diseño digital que permite crear prototipos interactivos y colaborar en el diseño.

  5. Zeplin

    6. Herramienta de colaboración que facilita el paso de los diseños de los diseñadores a los desarrolladores.

  6. GitHub

    7. Plataforma de desarrollo colaborativo que utiliza control de versiones para gestionar y compartir código.


Tema 2: El Diseño De Una Página Web

  1. Tipos de páginas Web

    2. 1. Página web estática: Contenido fijo que no cambia a menos que se edite manualmente. Ideal para pequeñas empresas o portafolios personales.

    2. Página web dinámica: Contenido que cambia en función de la interacción del usuario o de los datos de la base de datos. Ejemplos incluyen redes sociales, blogs y sitios de noticias.

    3. Página web de comercio electrónico: Diseñada para vender productos o servicios en línea. Incluye funcionalidades como carritos de compra, pasarelas de pago y catálogos de productos.

    4. Página web corporativa: Representa a una empresa u organización y proporciona información sobre su misión, productos, servicios y contacto. Puede incluir secciones como "Sobre nosotros", "Servicios", "Blog" y "Contacto".

    5. Portfolio: Mostrar el trabajo y proyectos de un individuo o empresa. Comúnmente utilizado por creativos como diseñadores, fotógrafos y escritores.

  1. Puntos básicos en la elaboración de una página Web.
  1. Planificación: Define propósito y objetivos.
  2. Diseño: Crea un diseño atractivo.
  3. HTML: Estructura básica de la página.
  4. CSS: Aplica estilos visuales.
  5. JavaScript: Añade interactividad.
  6. Pruebas: Verifica que funcione en todos los navegadores.
  7. Publicación: Sube la página y haz mantenimiento regular.

Tema 3: Estructura De Una Página Web

  1. Jerarquía de directorio de un proyecto web.

    2. Un proyecto web típicamente sigue una estructura de directorio organizada para mantener el código y los recursos de manera ordenada.

  1. proyecto-web/: Directorio raíz del proyecto.

  2. index.html: Archivo principal HTML.

  3. css/: Carpeta para archivos CSS.

  4. estilos.css: Archivo principal de estilos.

  5. js/: Carpeta para archivos JavaScript.

  6. scripts.js: Archivo principal de scripts.

  7. img/: Carpeta para imágenes.

  8. logo.png: Ejemplo de imagen.

  9. fondo.jpg: Ejemplo de imagen de fondo.

  10. fonts/: Carpeta para fuentes tipográficas.

  11. tipografia.ttf: Ejemplo de archivo de fuente.

  12. docs/: Carpeta para documentación.

  13. manual.pdf: Ejemplo de documento.

  14. README.md: Archivo de lectura inicial con información sobre el proyecto.

    Elementos de estructura para el diseño de una página Web: Concepto

  1. Regla utilizada para validar y certificar una página web

    2. Validación W3C: Es esencial validar una página web utilizando los estándares del World Wide Web Consortium (W3C). Esto asegura que el código HTML y CSS esté libre de errores y sea compatible con todos los navegadores. Las herramientas de validación W3C revisan el código y proporcionan informes detallados sobre cualquier problema encontrado.

  2. Reglas para escribir HTML.

    3. Se debe empezar identificando el lenguaje en el que vamos a programar, como es html tenemos que identificar con
    < !DOCTYPE html>, < header>, < main>, y demas, y las etiquetas que deben de ser cerradas cerrar con su misma etiqueta; Ej. < /header>

  3. < !DOCTYPE html>: Declara que el documento es HTML5. < html lang="es">: Define el idioma del contenido. < head>: Contiene metadatos, el título y enlaces a estilos. < body>: Contiene el contenido visible de la página. Estructura de contenido: Incluye < header>, < nav>, < main>, < section>, y < footer> para organizar el contenido de manera semántica.
